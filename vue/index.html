Vue.js を学ぶ

Vue.js の予習ノートです。手を動かす前に「言葉と全体像」だけ入れておくためのもの。 暗記は不要で、「リアクティブ」と「v-for / v-model」だけ頭の隅に残ればOK。 スマホでそのまま読めるノート全文(md)も置いています。

読む目的： あとで実際に Vue を触る前の地ならし。全体像を掴んでおくと、手を動かすときに迷いません。

① Vue.js ってなに（3行で）

  • 画面（HTML）を、データに合わせて自動で書き換えてくれる JavaScript の道具。
  • 素のJSは「ボタンが押された → 自分で文字を書き換える」を全部手書き。Vueはデータを変えるだけで画面が勝手に追従する。
  • Webフロントでよく使う。React と並ぶ代表的な選択肢のひとつ。

② 一番大事な考え方＝「リアクティブ」

Vue の肝はこれひとつ。値を変えると、その値を使っている画面が自動で更新されること。 下の2つを見比べると違いが分かります。

素のJS（手動で更新）

let count = 0; button.onclick = () => { count++; // ↓ 画面の更新を自分で書く display.textContent = count; };

Vue（自動で更新）

const count = ref(0); // 値を変えるだけ。画面はVueが更新 const increment = () => { count.value++; };
count を1増やすと、画面の {{ count }} の部分が勝手に1増える。 この「自動で追従」がリアクティブ。

③ 最小の動くコード（読むだけでOK）

ビルド環境なし。HTML に Vue を CDN で読み込むだけで動きます。

<div id="app"> <p>{{ message }}</p> <!-- {{ }} はデータが入る窓 --> </div> <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script> <script> const { createApp, ref } = Vue createApp({ setup() { const message = ref('こんにちは Vue') // ① リアクティブな値 return { message } // ③ returnして初めて使える } }).mount('#app') // ④ id="app" の場所に差し込む </script>
最小コードの読み方
書き方意味
ref('...')変わると画面が追従する値を作る。JS側で読み書きするときは message.value
{{ message }}HTML側にデータを表示する窓（マスタッシュ＝口ひげ）
return { ... }ここに入れた値だけが HTML 側で使える
.mount('#app')「この div を Vue が管理するよ」という宣言

④ ディレクティブ＝HTMLに付ける v-○○

HTMLタグに v-○○ という属性を付けて挙動を指定します。これをディレクティブと呼びます。 よく使う5つだけ覚えれば Todo は作れます。

よく使うディレクティブ5つ
書き方何をする短い例
v-bind:（略 :）属性にデータを流し込む<img :src="imgUrl">
v-on:（略 @）イベントを拾う<button @click="add">
v-model入力欄とデータを双方向で繋ぐ<input v-model="text">
v-for配列を繰り返し表示<li v-for="t in list">
v-if条件で出す/消す<p v-if="done">完了</p>
ざっくり：: は「渡す」、@ は「受け取る」。 v-model は「入力欄とデータが鏡みたいに同期」、 v-for は「リストを並べる」、v-if は「条件で出し分け」。

⑤ Todoアプリの完成イメージ（読むだけ）

「こういう形か」と眺めるだけでOK。④までの部品がぜんぶ入っています。

<div id="app"> <input v-model="newText" @keyup.enter="add" placeholder="やること" /> <button @click="add">追加</button> <ul> <li v-for="(todo, i) in todos" :key="i"> <input type="checkbox" v-model="todo.done" /> <span :class="{ done: todo.done }">{{ todo.text }}</span> <button @click="remove(i)">削除</button> </li> </ul> </div> <script> const { createApp, ref } = Vue createApp({ setup() { const newText = ref('') const todos = ref([]) // 配列もリアクティブ const add = () => { if (newText.value === '') return todos.value.push({ text: newText.value, done: false }) newText.value = '' // 入力欄を空に戻す } const remove = (i) => { todos.value.splice(i, 1) } return { newText, todos, add, remove } } }).mount('#app') </script>
  • v-for="(todo, i) in todos" … 配列を1件ずつ todo として並べる。i は番号（削除に使う）。
  • v-model="todo.done" … チェックボックスと done(true/false)が同期。
  • :class="{ done: todo.done }" … done が true のときだけ done クラスを付ける（打ち消し線など）。
  • todos.value.push(...) … 配列に足すと v-for の表示も自動で増える（＝リアクティブ）。

⑥ 面接で言えること（正直版・盛らない）

  • 「説明会で Vue.js を使うと伺ったので、公式チュートリアルを触って、簡単な Todo アプリを作ってみました。」
  • 「素のJSだと画面の更新を自分で書きますが、Vueはデータを変えると画面が自動更新（リアクティブ）なのが新鮮でした。」
  • 「v-for でリスト表示、v-model で入力との同期、といった基本を試した段階です。これから実務で深めたいです。」
ポイント：「もう触った」事実 ＋ 正直に「基本の段階」と言う。背伸びしないのが一番効きます。
