JavaScript 超入門

Vueの予習ノートを読んで「言いたいことは分かるけど、コードの一行一行が読めない…」となった人向け。 JavaScript（JS）をゼロから、一段階ずつ積み上げます。暗記は不要。 ゴールはVueの最小コードとTodoコードを、自分で一行ずつ訳せるようになること。 スマホでそのまま読めるノート全文(md)も置いています。

読み方： 上から順に「段階（ステージ）」で積み上げます。コードは全部日本語に訳す感覚でOK。 試したくなったらブラウザで F12 → コンソールにそのまま打てます。

段階0. JavaScript ってそもそも何？

Webページを「動かす」ための言語。Webの3兄弟で役割分担しています。

Webの3兄弟の役割
言語役割たとえ
HTML骨組み・中身（見出し・文章・ボタン）家の構造
CSS見た目（色・大きさ・配置）内装・塗装
JavaScript動き（押したら数字が増える等）電気配線
JSはブラウザの中で動くので、特別なインストールは不要。ブラウザさえあれば動きます。

段階1. 変数 ＝ 値に名前をつける

値（数値や文字）に名前をつけて後から呼び出せるようにするのが変数です。

let age = 20;
  • let … 「これから変数を作る」の宣言。
  • age … 自分でつけた名前（中身が分かる名前に）。
  • = … 右の値を左に入れる（「等しい」ではない）。
  • 20 … 入れる値。
  • ; … 一文の終わり。1命令＝1セミコロン。

let と const の違い（超重要）

let count = 0; // あとで書き換えOK const name = '太郎'; // 書き換え禁止（定数） count = 1; // ✅ OK name = '次郎'; // ❌ エラー（constは再代入できない）
迷ったらまず const。「これは変える」と分かってから let に。 Vueのコードもほぼ const です。

段階2. 値の種類（データ型）

Vueを読むのに必要な5つだけ。

値の種類5つ
種類何例
文字列文字（クォートで囲む）'こんにちは'
数値数（クォート無し・計算できる）20 / 19.8
真偽値はい/いいえの2つだけtrue / false
配列順番に並べたリスト（0始まり）['りんご','みかん']
オブジェクト名前つきの値の集まり{ text:'牛乳', done:false }
const fruits = ['りんご', 'みかん', 'ぶどう']; fruits[0] // 'りんご'（番号は0から！） fruits[1] // 'みかん' const todo = { text: '牛乳を買う', done: false }; todo.text // '牛乳を買う'（.キー名 で取り出す） todo.done // false
この5つが分かれば、Vueのコードに出てくる値は全部読めます。 配列は同じ種類をたくさん、オブジェクトは関連情報を1個にまとめるとき。

段階3. 関数 ＝ 処理に名前をつける

「やること（処理）のまとまり」に名前をつけ、何度でも呼べるようにするのが関数です。

昔ながらの書き方

function add(a, b) { return a + b; } add(2, 3); // → 5

いまどき＝アロー関数（Vueはこっち）

const add = (a, b) => { return a + b; }; add(2, 3); // → 5
  • (a, b) … 受け取る材料（引数）。呼ぶときに渡す。
  • => … アロー（矢印）。「引数 → 処理」をつなぐアロー関数の目印。
  • return … この値を答えとして返す。
  • 引数なしのときは () だけ書く（空でもカッコは必要）。
Vueの const increment = () => { count.value++ } も、この引数なしのアロー関数です。

段階4. 条件分岐（if）と比較（===）

「もし〜なら、〜する」を書くのが if。

const age = 20; if (age >= 18) { console.log('大人です'); } else { console.log('子どもです'); }
比較の記号
記号意味例
===等しい（厳密）3 === 3 → true
!==等しくない3 !== 4 → true
> >=より大きい / 以上5 > 3 → true
< <=より小さい / 以下2 <= 2 → true
「等しい」は ===（イコール3つ）。 = 1つは「代入（入れる）」で意味が全然違う。ここ混同しやすい。

関数の中で「条件に合わなければ何もせず終了」したいとき：

const add = () => { if (newText === '') return; // 空なら、ここで終了 console.log('追加する処理'); // 通過したときだけ動く };
return;（値なし）は「ここで関数を終わり」の合図。 これが Todo の if (newText.value === '') return の正体です。

段階5. 配列の操作（push / splice）

配列を増やす・減らす方法。Todoの追加・削除がこれです。

const list = ['りんご', 'みかん']; list.push('ぶどう'); // → ['りんご','みかん','ぶどう']（末尾に追加） const list2 = ['りんご', 'みかん', 'ぶどう']; list2.splice(1, 1); // → ['りんご','ぶどう']（1番目から1個削除）
  • list.push(値) … 末尾に1つ足す。
  • list.splice(開始番号, 消す個数) … 何番目から何個消すか（番号は0始まり）。
  • .push() のように値に付ける「.命令()」をメソッドと呼ぶ。

配列の中身はオブジェクトでもOK。これがTodoリストの形：

const todos = []; todos.push({ text: '牛乳を買う', done: false }); todos.push({ text: '掃除する', done: true }); // 1件＝オブジェクト、全体＝オブジェクトの配列

段階6. オブジェクトをもう少し（点でつなぐ .）

const todo = { text: '牛乳を買う', done: false }; todo.text // '牛乳を買う'（読む） todo.done = true; // 書き換える（false → true）
. の意味は一貫して「左のものが持っている、右のもの」。
todo.text（todoが持つtext）／ list.push()（listが持つpush）／ count.value（countが持つvalue・Vueで出る）。

段階7. 画面を触る（DOM）とイベント

ここが素のJSのハイライト。Vueがなぜ便利かの土台です。 ブラウザは読み込んだHTMLを、JSから操作できる形（＝DOM）で持っています。

<p id="output">0</p> <button id="btn">+1</button> const output = document.getElementById('output'); // 要素を取ってくる const button = document.getElementById('btn'); let count = 0; button.onclick = () => { // クリックされたら実行 count++; // データを変える output.textContent = count; // ← 画面を自分で書き換える };
  • document.getElementById('output') … idで要素を1つ取ってくる。
  • button.onclick = 関数 … クリックされたら関数を動かすと登録。
  • count++ … 1増やす（count = count + 1 の短縮）。
  • textContent … 要素の表示テキスト。ここに入れて表示更新。
核心：素のJSは「①データを変える」「②画面を書き換える」の2手が常に必要。 表示が10箇所変わるなら10箇所手書き…これが大変。 Vueは②を自動でやる（＝リアクティブ）。Vueノートの「素のJS vs Vue」はこの話でした。

段階8. 分割代入（const { a, b } = obj）

オブジェクトから必要なキーを取り出して、同じ名前の変数にする書き方。Vueの1行目に出ます。

const person = { name: '太郎', age: 20, city: '東京' }; // ふつうに取り出すと… const name = person.name; const age = person.age; // 分割代入なら1行で同じこと const { name, age } = person; // name='太郎', age=20
Vueの1行目 const { createApp, ref } = Vue は、 「Vue という道具箱から createApp と ref を取り出して使えるようにする」という意味です。

段階10. Vue最小コードを一行ずつ訳す

新出はVue特有の2つ（ref と .value）だけ。あとは段階1〜8の組み合わせです。

<div id="app"> <p>{{ message }}</p> </div> <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script> <script> const { createApp, ref } = Vue createApp({ setup() { const message = ref('こんにちは Vue') return { message } } }).mount('#app') </script>
  • <div id="app"> … Vueが面倒を見る縄張り。id="app" は名指し用の名札。
  • {{ message }} … Vue専用のデータを表示する窓。素のJSの textContent の代わり。
  • <script src=...vue...> … Vue本体をネット経由で読み込む（CDN）。これで Vue 道具箱が使える。
  • const { createApp, ref } = Vue … 段階8の分割代入。2つの道具を取り出す。
  • createApp({ ... }) … アプリを1つ作る。中の { } は「設定一式」のオブジェクト。
  • setup() { ... } … 起動時に呼ばれる関数。使うデータと処理を準備する場所。
  • ★const message = ref('こんにちは Vue') … ref＝変わると画面が追従する値を作る。
  • return { message } … return した値だけがHTMLの {{ }} で使える（返し忘れ注意）。
  • .mount('#app') … 作ったアプリを id="app" の場所に取り付ける（#＝idが〜）。
★Vue新出②：ref の値をJS側で読み書きするときは .value を付ける（{{ }}内では不要）。
const count = ref(0); count.value; // 0（読む） count.value++; // 1（書く）
段階6「. は持ち物」がそのまま効いています（countが持つvalue）。覚えたVue語はたった2つだけ。

段階11. Todoコードを一行ずつ訳す

最後。新出ゼロ、全部これまでの組み合わせです。

const { createApp, ref } = Vue createApp({ setup() { const newText = ref('') const todos = ref([]) const add = () => { if (newText.value === '') return todos.value.push({ text: newText.value, done: false }) newText.value = '' } const remove = (i) => { todos.value.splice(i, 1) } return { newText, todos, add, remove } } }).mount('#app')
  • const newText = ref('') … 入力欄の文字を入れるリアクティブな値。最初は空。
  • const todos = ref([]) … Todo一覧のリアクティブな配列。増減すると画面も自動で増減。
  • const add = () => { … 追加ボタンの処理（引数なしのアロー関数）。
  • if (newText.value === '') return … 段階4の門番。空なら終了。
  • todos.value.push({ text: newText.value, done: false }) … 配列に1件のTodo（オブジェクト）を追加。画面のリストも自動で増える。
  • newText.value = '' … 入力欄を空に戻す。画面の入力欄も自動で空に。
  • const remove = (i) => { todos.value.splice(i, 1) } … i番目を1個削除（番号は0始まり）。
  • return { newText, todos, add, remove } … HTMLで使う値と処理をまとめて返す。

HTML側のディレクティブも訳す

  • v-model="newText" … 入力欄と newText を双方向で同期。
  • @keyup.enter="add" … 入力欄でEnterを離したら add を実行（@＝受け取る）。
  • v-for="(todo, i) in todos" … 配列を1件ずつ繰り返し表示（各件 todo、番号 i）。
  • v-model="todo.done" … チェックボックスと todo.done(true/false)を同期。
  • :class="{ done: todo.done }" … done が true のときだけ done クラスを付ける（:＝渡す）。
  • @click="remove(i)" … 削除ボタンで remove(i) を実行（i番目を消す）。
全部、段階1〜8の文法＋ref / .value / ディレクティブだけ。 新しい難しさはありません。

おわりに ＆ 次の一歩

  • プログラムは「コンピュータへの指示文」。一行ずつ日本語に訳せば必ず読める。
  • 詰まったら段階9のチェックリストに戻る。だいたいそこに答えがあります。
  • 次の一歩：ブラウザのコンソール（F12）で短いコードを実際に打つ。動くと一気に腹落ちします。
