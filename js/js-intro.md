## 段階0. JavaScript ってそもそも何？

- **Webページを「動かす」ための言語**です。
- ざっくり3兄弟で役割分担：
  - **HTML** … ページの「骨組み・中身」（見出し、文章、ボタンを置く）
  - **CSS** … ページの「見た目」（色、大きさ、配置）
  - **JavaScript** … ページの「動き」（ボタンを押したら数字が増える、など）
- JSは**ブラウザの中で動く**。だから特別なインストールは不要で、ブラウザさえあれば動きます。

> イメージ：HTMLが「家の構造」、CSSが「内装・塗装」、JSが「電気配線（押したら明かりがつく）」。

## 段階1. 変数 ＝ 値に名前をつける

プログラムは、数値や文字といった「値」を扱います。
値に**名前をつけて、後から呼び出せるようにする**のが**変数**です。

```js
let age = 20;
```

これを一行ずつ訳すと：

- `let` … 「これから**変数を作る**よ」という宣言の言葉（キーワード）。
- `age` … 自分でつけた**名前**（変数名）。なんでもいいが、中身が分かる名前にする。
- `=` … 「**右の値を左に入れる**」という意味。**等しい、ではない**ので注意（数学の=と違う）。
- `20` … 入れる**値**。
- `;` … 「この一文はここで終わり」という区切り。**1命令＝1セミコロン**が基本。

まとめると「`age` という名前の箱を作って、中に `20` を入れた」。

### `let` と `const` の違い（ここ超重要）

```js
let count = 0;     // あとで中身を変えられる
const name = '太郎'; // あとで中身を変えられない（再代入できない）
```

- `let` … **あとで書き換える**予定の値に使う。（例：カウンターの数）
- `const` … **書き換えない**値に使う。（const = constant＝定数）
- **迷ったらまず `const`**。「これは変える」と分かってから `let` にする、が今どきの基本。

```js
let count = 0;
count = 1;   // ✅ OK。letだから書き換えられる

const name = '太郎';
name = '次郎'; // ❌ エラー。constは書き換え禁止
```

> Vueのコードでも `const message = ...` のように **ほぼ const** が出てきます。「基本const」を覚えておけば読めます。

## 段階2. 値の種類（データ型）

値にはいくつか「種類」があります。Vueを読むのに必要な5つだけ。

### ① 文字列（String）＝ 文字。クォートで囲む

```js
const greeting = 'こんにちは';
const name2 = "太郎";  // ' でも " でもOK（どちらかで囲めばよい）
```

- 文字は必ず **`'` か `"` で囲む**。囲まないと「変数名」と勘違いされてエラーになる。

### ② 数値（Number）＝ 数。クォートは付けない

```js
const age = 20;      // 整数
const price = 19.8;  // 小数もOK
```

- 計算ができる：`age + 1` → `21`。
- 文字列の `'20'` と 数値の `20` は**別物**。`'20'` は計算用じゃない文字。

### ③ 真偽値（Boolean）＝ はい/いいえ。値は2つだけ

```js
const isDone = true;   // 本当（はい）
const isOpen = false;  // 嘘（いいえ）
```

- `true` か `false` の**2つしかない**。
- 「終わった？」「開いてる？」のような **YES/NO** を表す。Todoの「完了したか」で使います。

### ④ 配列（Array）＝ 値を順番に並べたリスト。`[ ]` で囲む

```js
const fruits = ['りんご', 'みかん', 'ぶどう'];
```

- `[ ]` の中に、値を `,` で区切って並べる。
- **順番に番号（インデックス）がつく。番号は 0 から始まる**（ここ大事）。
  - `fruits[0]` → `'りんご'`（1番目）
  - `fruits[1]` → `'みかん'`（2番目）
  - `fruits[2]` → `'ぶどう'`（3番目）
- 「Todoの一覧」のように**同じ種類のものをたくさん**持つときに使う。

### ⑤ オブジェクト（Object）＝ 名前つきの値の集まり。`{ }` で囲む

```js
const todo = { text: '牛乳を買う', done: false };
```

- `{ 名前: 値, 名前: 値 }` の形。名前のことを **キー（key）**、中身を **値（value）** と呼ぶ。
- 取り出すときは **`.キー名`**：
  - `todo.text` → `'牛乳を買う'`
  - `todo.done` → `false`
- 「1件のTodo（文章＋完了フラグ）」のように **関連する情報をまとめて1個**にするときに使う。
- 詳しくは段階6でもう一度やります。

> この5つ（文字列・数値・真偽値・配列・オブジェクト）が分かれば、Vueのコードに出てくる値は全部読めます。

## 段階3. 関数 ＝ 処理に名前をつける

**「やること（処理）のまとまり」に名前をつけて、何度でも呼び出せる**ようにするのが**関数**です。

### まず昔ながらの書き方（function）

```js
function add(a, b) {
  return a + b;
}
```

一行ずつ訳す：

- `function` … 「**関数を作る**よ」の宣言。
- `add` … 関数の**名前**。
- `(a, b)` … **受け取る材料（引数＝ひきすう）**。呼ぶときに渡す。
- `{ ... }` … `{` から `}` までが**処理の中身**（ブロック）。
- `return a + b;` … `return` は「**この値を答えとして返す**」。ここでは `a + b` の結果を返す。

使うとき（呼び出し）：

```js
const result = add(2, 3);  // a=2, b=3 として実行 → 5 が返る
// result の中身は 5
```

### いまどきの書き方＝アロー関数 `=>`

Vueのコードはほぼ**こっち**で書かれています。同じ「add」をアロー関数で書くと：

```js
const add = (a, b) => {
  return a + b;
};
```

訳す：

- `const add =` … 「`add` という名前に、**関数そのものを値として入れる**」。関数も値として変数に入れられる。
- `(a, b)` … 引数（受け取る材料）。
- `=>` … 「アロー（矢印）」。**「引数 → 処理」をつなぐ記号**。これがアロー関数の目印。
- `{ ... }` … 処理の中身。

### アロー関数の短縮形（Vueで超頻出）

処理が「1行で値を返すだけ」なら、`{ }` と `return` を省略できる：

```js
const add = (a, b) => a + b;   // {} と return を省略。a+b の結果がそのまま返る
```

引数がないときは `()` だけ書く：

```js
const sayHi = () => {
  console.log('やあ');
};
```

- `()` … **引数なし**でも空のカッコは必要。
- `console.log('やあ')` … `console.log(...)` は「**カッコの中身をコンソールに表示する**」命令。動作確認の定番。

> Vueの `const increment = () => { count.value++ }` も、この「引数なしのアロー関数」です。もう半分読めます。

## 段階4. 条件分岐（if）と比較（===）

「**もし〜なら、〜する**」を書くのが `if` です。

```js
const age = 20;

if (age >= 18) {
  console.log('大人です');
} else {
  console.log('子どもです');
}
```

訳す：

- `if (条件) { ... }` … カッコの**条件が true のとき**だけ `{ }` の中を実行。
- `age >= 18` … 「age は 18 以上か？」という**比較**。結果は `true` か `false`（段階2の真偽値！）。
- `else { ... }` … 条件が false のとき実行する側。

### 比較の記号

| 記号 | 意味 | 例（結果） |
| --- | --- | --- |
| `===` | 等しい（厳密） | `3 === 3` → true |
| `!==` | 等しくない | `3 !== 4` → true |
| `>` `>=` | より大きい / 以上 | `5 > 3` → true |
| `<` `<=` | より小さい / 以下 | `2 <= 2` → true |

- **「等しい」は `===`（イコール3つ）**。`=` 1つは「代入（入れる）」なので意味が全然違う。ここ混同しやすい。
- `==`（イコール2つ）もあるが、今は **`===` を使う** と覚えればOK（より安全なため）。

### 早めに終わらせる `return`（Vueで出ます）

関数の中で「条件に合わなければ、もう何もせず終了」したいとき：

```js
const add = () => {
  if (newText === '') return;  // 空っぽなら、ここで関数を終了（下を実行しない）
  // ↑ここを通過したときだけ、下の処理が動く
  console.log('追加する処理');
};
```

- `return;`（値なし）は「**ここで関数を終わり**」の合図。
- 「変な入力なら早めに切り上げる」という**門番**としてよく使う。これが Todo の `if (newText.value === '') return` の正体。

## 段階5. 配列の操作（push / splice）

段階2の配列を、**増やしたり減らしたり**する方法です。Todoの追加・削除がこれ。

### 末尾に追加 ＝ push

```js
const list = ['りんご', 'みかん'];
list.push('ぶどう');
// list は ['りんご', 'みかん', 'ぶどう'] になる
```

- `list.push(値)` … 「`list` の**末尾に値を1つ足す**」。
- `.push(...)` のように、**値（ここでは配列）に対して `.命令名()`** を付けると、その値専用の操作ができる。この「.なんとか()」を**メソッド**と呼ぶ。

### 指定位置から削除 ＝ splice

```js
const list = ['りんご', 'みかん', 'ぶどう'];
list.splice(1, 1);
// 1番目（みかん）から 1個 削除 → ['りんご', 'ぶどう']
```

- `list.splice(開始番号, 消す個数)` … 「**何番目から、何個消すか**」。
- `splice(1, 1)` は「番号1（＝2番目のみかん）から1個消す」。番号は0始まりを思い出して。
- Todoの「削除ボタン」は `todos.splice(i, 1)` ＝「i番目を1個消す」でできています。

### オブジェクトを配列に足す（Todoの本体）

配列の中身は文字だけでなく、**オブジェクトでもOK**：

```js
const todos = [];                                 // 空の配列
todos.push({ text: '牛乳を買う', done: false });   // オブジェクトを1件追加
todos.push({ text: '掃除する',   done: true  });
// todos は [{text:'牛乳を買う',done:false}, {text:'掃除する',done:true}]
```

- これが「Todoリスト」のデータの形。**1件＝オブジェクト、全体＝オブジェクトの配列**。

## 段階6. オブジェクトをもう少し（点でつなぐ `.`）

段階2で出たオブジェクトを、読み書きする練習。

```js
const todo = { text: '牛乳を買う', done: false };

console.log(todo.text);   // '牛乳を買う'  ← 読む
console.log(todo.done);   // false

todo.done = true;         // 書き換える（false → true）
console.log(todo.done);   // true
```

- **`.キー名` で中身にアクセス**できる（読むのも書くのも `.` 経由）。
- `todo.done = true` は段階1の「`=` は代入」と同じ。`done` の中身を `true` に入れ替えた。
- Todoの「チェックを入れたら完了になる」は、この `todo.done` を true/false で切り替えているだけ。

### `.` は「持ち物をたどる」記号

`.` は色々な場面で出ますが、意味は一貫して **「左のものが持っている、右のもの」**：

- `todo.text` … todo が持つ text
- `list.push()` … list が持つ push という命令
- `count.value` … count が持つ value （← Vueで出る。後述）

> 「`.` の左が持ち主、右が中身/命令」とだけ覚えれば、知らないコードでも構造が読めます。

## 段階7. 画面を触る（DOM）とイベント

ここが「**素のJS**」のハイライト。Vueがなぜ便利かを理解する土台です。

### DOM ＝ JSから見たHTML

ブラウザは、読み込んだHTMLを**JSから操作できる形**にして持っています。これを **DOM** と呼びます。
JSは「HTMLのこの部分」を**取ってきて、書き換える**ことができます。

```html
<p id="output">0</p>
<button id="btn">+1</button>
```

```js
const output = document.getElementById('output'); // id="output" の<p>を取ってくる
const button = document.getElementById('btn');     // id="btn" の<button>を取ってくる
```

- `document` … 「いま表示しているページ全体」を指す、最初から用意された箱。
- `document.getElementById('output')` … 「**idが output の要素を1つ取ってくる**」命令。
- 取ってきた要素を変数（`output` / `button`）に入れて、後で触れるようにした。

### イベント ＝「〜されたら、〜する」

```js
let count = 0;

button.onclick = () => {   // ボタンが「クリックされたら」、この関数を実行
  count++;                 // count を1増やす（count = count + 1 の短縮）
  output.textContent = count; // ← 画面の文字を、自分で書き換える
};
```

一行ずつ：

- `button.onclick = 関数` … 「button が**クリックされたら、この関数を動かす**」と登録。
- `() => { ... }` … 段階3でやった引数なしのアロー関数。これが「クリック時にやること」。
- `count++` … `count` を1増やす。`++` は「1足す」の短縮（`count = count + 1` と同じ）。
- `output.textContent = count` … `textContent` は「その要素の**表示テキスト**」。そこに count を入れて**表示を更新**。

### ここが核心：素のJSは「画面の更新を手書き」

注目は最後の行。**count を増やしただけでは画面は変わりません**。
`output.textContent = count` と**自分で書いて初めて**画面が変わります。

> つまり素のJSでは「①データを変える」「②画面を書き換える」の**2手**が常に必要。
> Todoが10個あって10箇所表示が変わるなら、その10箇所ぜんぶ手で書く…これが大変。
> **Vueは②を自動でやってくれる**（＝リアクティブ）。Vueノートの「素のJS vs Vue」の比較は、まさにこの話でした。

## 段階8. 分割代入（const { a, b } = obj）

Vueの1行目に出てくる、最初の関門がこれ。先に正体を知っておきます。

オブジェクトから**必要なキーを取り出して、同じ名前の変数にする**書き方です。

```js
const person = { name: '太郎', age: 20, city: '東京' };

// ふつうに取り出すと…
const name = person.name;
const age = person.age;

// 分割代入なら1行で同じことができる
const { name, age } = person;
// → name に '太郎'、age に 20 が入る（cityは取り出していないので無視）
```

訳す：

- `const { name, age } = person` … 「person の中から **name と age を取り出して、それぞれ同じ名前の変数に入れる**」。
- 左辺の `{ }` は「オブジェクトを作る」ではなく「**取り出す**」の意味。形が似てるので最初は戸惑うが、`= 右側` から来るときは「取り出し」と覚える。

Vueの1行目：

```js
const { createApp, ref } = Vue
```

- これは「`Vue`（CDNで読み込んだ道具箱）の中から、**createApp と ref という2つの道具を取り出して使えるようにする**」という意味。
- `Vue` という大きな箱の中に色々入っていて、その中から今回使う2つだけ抜き出した、ということ。

## 段階9. ここまでの総まとめ（チェックリスト）

Vueを読む前に、これが「あー、はいはい」と言えればOK：

| 記号・言葉 | 意味（ひとことで） |
| --- | --- |
| `const` / `let` | 値に名前をつける（const＝変えない / let＝変える） |
| `'...'` `"..."` | 文字列（文字） |
| `[ ... ]` | 配列（順番つきのリスト、0始まり） |
| `{ key: value }` | オブジェクト（名前つきの値の集まり） |
| `() => { ... }` | アロー関数（処理のまとまり） |
| `return` | 関数の答えを返す／早めに終了 |
| `if (条件)` | 条件が true のとき実行 |
| `===` | 等しい（`=` は代入なので別物） |
| `.push()` `.splice()` | 配列に足す／消す |
| `.キー` | オブジェクトの中身にアクセス（`.`＝持ち物） |
| `document.getElementById` | HTMLの要素を取ってくる |
| `要素.onclick = 関数` | クリックされたら関数を動かす |
| `const { a, b } = obj` | オブジェクトから取り出す（分割代入） |

ここから先は、**これらの組み合わせ**でしかありません。新しい難しい文法は出てきません。

## 段階10. Vue最小コードを一行ずつ訳す

ではVueノートのコードを、いまの知識で訳していきます。**新出はVue特有の2つ（`ref` と `.value`）だけ**。

```html
<div id="app">
  <p>{{ message }}</p>
</div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp, ref } = Vue

  createApp({
    setup() {
      const message = ref('こんにちは Vue')
      return { message }
    }
  }).mount('#app')
</script>
```

### HTML側

- `<div id="app">` … Vueに管理してもらう**縄張り**。この中だけVueが面倒を見る。`id="app"` は後で名指しするための名札。
- `<p>{{ message }}</p>` … `{{ }}` は **Vue専用の「データを表示する窓」**（マスタッシュと呼ぶ）。`message` という値をここに表示する、という意味。素のJSの `textContent` の代わり、と思えばよい。

### `<script src=...>`

- `<script src="...vue.global.js"></script>` … **Vue本体をネット経由で読み込む**行（CDN）。これを読み込むと、段階8で出た `Vue` という道具箱が使えるようになる。

### JS側を一行ずつ

```js
const { createApp, ref } = Vue
```
- 段階8でやった**分割代入**。`Vue` から `createApp`（アプリを作る道具）と `ref`（リアクティブな値を作る道具）を取り出した。

```js
createApp({ ... })
```
- `createApp(...)` … 「**Vueアプリを1つ作る**」命令。
- カッコの中の `{ ... }` は段階2の**オブジェクト**。「このアプリの設定一式」を1個のオブジェクトで渡している。

```js
    setup() {
```
- `setup` … そのオブジェクトの中の**キー**で、中身は**関数**（`setup() { ... }` は「setupという名前の関数」の書き方）。
- Vueは起動時にこの `setup` 関数を呼ぶ。「**この画面で使うデータと処理を、ここで準備してね**」という場所。

```js
      const message = ref('こんにちは Vue')
```
- ★Vue新出①：`ref(値)` … 「**変わると画面が自動で追従する値**」を作る道具。
- `ref('こんにちは Vue')` で「こんにちは Vue」という文字を、**リアクティブな入れ物に包んで** message に入れた。
- ふつうの `const message = 'こんにちは Vue'` との違いは「**ただの文字か / 画面と連動する文字か**」だけ。

```js
      return { message }
```
- 段階3の `return`。`setup` 関数の**答えとして、使ってほしい値をオブジェクトで返す**。
- `{ message }` は `{ message: message }` の短縮形（キーと変数名が同じときは1個で書ける）。
- **ここで return した値だけ**が、HTMLの `{{ }}` で使える。返し忘れると画面に出ない、は初心者あるある。

```js
  }).mount('#app')
```
- `.mount('#app')` … 「作ったアプリを、**`id="app"` の場所に取り付ける**」。`#app` の `#` は「idが〜」という指定（CSSと同じ書き方）。
- これでHTMLの `<div id="app">` の中をVueが管理し始め、`{{ message }}` に「こんにちは Vue」が表示される。

### `.value` はどこ？（リアクティブ値の読み書き）

`ref` で作った値を**JS側で読み書きするときは `.value` を付ける**ルールがあります（HTMLの `{{ }}` 内では不要）。

```js
const count = ref(0);
console.log(count.value); // 0  ← 読むときも .value
count.value++;            // 1  ← 書くときも .value
```

- `count` は「リアクティブな箱」で、**中身は `count.value` にある**。だから読み書きは `.value` 経由。
- 段階6の「`.` は持ち物をたどる」がそのまま効いています（count が持つ value）。
- Vueノートの `count.value++`（カウンター）や `newText.value`（Todo）はこれです。

> 新しく覚えたVue語はたった2つ：**`ref(値)`** と **`.value`**。あとは段階1〜8の組み合わせ。

## 段階11. Todoコードを一行ずつ訳す

最後。Vueノート§5のTodoコードを、全部いまの知識で訳します。新出ゼロ（全部これまでの組み合わせ）。

```js
const { createApp, ref } = Vue
createApp({
  setup() {
    const newText = ref('')
    const todos = ref([])
    const add = () => {
      if (newText.value === '') return
      todos.value.push({ text: newText.value, done: false })
      newText.value = ''
    }
    const remove = (i) => { todos.value.splice(i, 1) }
    return { newText, todos, add, remove }
  }
}).mount('#app')
```

一行ずつ：

```js
const { createApp, ref } = Vue
```
- §10と同じ。道具箱から createApp と ref を取り出す。

```js
    const newText = ref('')
```
- 入力欄の文字を入れる**リアクティブな値**。`''`（空の文字列）から始める＝最初は空っぽ。

```js
    const todos = ref([])
```
- Todo一覧を入れる**リアクティブな配列**。`[]`（空の配列）から始める＝最初はTodoゼロ件。
- 配列もリアクティブにできるので、**増減すると画面のリストも自動で増減**する。

```js
    const add = () => {
```
- 「追加ボタンを押したときの処理」を `add` という名前の**アロー関数**で用意（段階3）。

```js
      if (newText.value === '') return
```
- 段階4の**門番**。入力欄（`newText.value`）が**空なら、ここで終了**（空のTodoを作らせない）。
- `newText.value` ＝ refの中身を読む（段階10の `.value`）。`=== ''` ＝ 空文字と等しいか（段階4）。

```js
      todos.value.push({ text: newText.value, done: false })
```
- ここが追加の本体。分解すると：
  - `todos.value` … refの中身（配列本体）を取り出す。
  - `.push( ... )` … その配列の末尾に追加（段階5）。
  - `{ text: newText.value, done: false }` … 追加する**1件のTodo（オブジェクト）**。
    - `text` に入力された文字（`newText.value`）、`done` に `false`（まだ未完了）。
- 配列に1件足すと、リアクティブなので**画面のリストにも自動で1行増える**。

```js
      newText.value = ''
```
- 追加し終わったら入力欄を**空に戻す**（次の入力に備える）。`= ''` で空文字を代入（段階1）。
- これもリアクティブなので、画面の入力欄も自動で空になる。

```js
    const remove = (i) => { todos.value.splice(i, 1) }
```
- 「削除処理」。`i`（消したい番号）を**引数で受け取る**アロー関数。
- `todos.value.splice(i, 1)` … 配列の **i番目から1個削除**（段階5）。番号は0始まり。
- 削除すると、画面のリストからもその行が自動で消える。

```js
    return { newText, todos, add, remove }
```
- 段階3の `return`。**HTMLで使う値と処理を、まとめて返す**。
- `{ newText, todos, add, remove }` は短縮形。これで `{{ }}` や `@click="add"` から使えるようになる。

```js
  }).mount('#app')
```
- §10と同じ。アプリを `id="app"` に取り付けて完成。

### HTML側のディレクティブも訳す

```html
<input v-model="newText" @keyup.enter="add" placeholder="やること" />
```
- `v-model="newText"` … 入力欄と `newText` を**双方向で同期**（入力すれば newText に入り、newText を変えれば入力欄も変わる）。
- `@keyup.enter="add"` … `@` は「イベントを受け取る」。**入力欄でEnterキーを離したら `add` を実行**。
- `placeholder` … 素のHTMLの属性。空のときに薄く出る説明文。

```html
<li v-for="(todo, i) in todos" :key="i">
```
- `v-for="(todo, i) in todos"` … 配列 `todos` を**1件ずつ繰り返し表示**。各件を `todo`、番号を `i` として取り出す（段階5の「配列をたどる」が画面で起きるイメージ）。
- `:key="i"` … Vueが各行を見分けるための目印（つけるお作法、と今は思ってOK）。

```html
<input type="checkbox" v-model="todo.done" />
<span :class="{ done: todo.done }">{{ todo.text }}</span>
<button @click="remove(i)">削除</button>
```
- `v-model="todo.done"` … チェックボックスと `todo.done`（true/false）を同期（段階6の `.done` 書き換え）。
- `:class="{ done: todo.done }"` … `todo.done` が true のときだけ `done` というCSSクラスを付ける（打ち消し線などに使う）。`:` は「渡す」。
- `{{ todo.text }}` … そのTodoの文章を表示。
- `@click="remove(i)"` … 削除ボタンを押したら `remove(i)` を実行（i番目を消す）。

> ここまで全部、**段階1〜8で出た文法の組み合わせ**＋ **ref / .value / ディレクティブ** だけ。新しい難しさはありません。
