# yuki's atelier

日々学んだことを**ジャンルごとにまとめていく、自分専用の学習サイト**です。
ルートの `index.html` がハブ（入口）で、各ジャンルはフォルダに分けて置きます。
デザイン（`css/style.css`）は全ジャンル共通、**問題形式や解説画面はジャンルごとに専用**で作っていく方針です。

- **スマホ優先・PCでも快適**：スマホでの読みやすさを基準に作り、広い画面で自動拡張
- **ビルド不要**：`index.html` をブラウザで開くだけ（相対パスのみ）
- **少しずつ増やす**：学んだその日に項目を足せるシンプル構成

### 現在のジャンル

| ジャンル | 場所 | 内容 |
| --- | --- | --- |
| **Git / GitHub** | [`git/`](git/index.html) | 基本コマンドを“状態の変化”で学ぶ。シミュレーター・クイズ・ブランチ／チーム開発の解説つき |

## Git ジャンル（`git/`）

Gitの基本コマンドを「文章の暗記」ではなく、**作業フォルダ・ステージング・ローカルリポジトリ・GitHub の“状態の変化”** として直感的に学べます。

`git add` / `git commit` / `git push` などを実行すると、ファイルカードやコミットカードが4つのエリア間を移動し、Gitが裏で何をしているのかを目で見て理解できます。

> ⚠️ 本物のGitは動きません。すべて学習用の**疑似デモ（シミュレーター）**です。安心して何度でも試せます。

---

## サイトの目的

- Git初心者が `status / add / commit / push / pull` などの基本操作を理解する
- 「どのコマンドが、どの場所を、どう動かすのか」を**状態変化**として掴む
- 状況（やりたいこと）からコマンドを引けるようにする
- クイズと復習で定着させる

---

## 4つの状態エリア

| エリア | 役割 |
| --- | --- |
| 作業フォルダ | あなたがファイルを編集する場所 |
| ステージング | 次のコミットに含める準備をする控え室 |
| ローカルリポジトリ | 手元に記録（コミット）が保存される場所 |
| GitHub（リモート） | みんなで共有する場所 |

`git add` で 作業フォルダ → ステージング、`git commit` で ステージング → ローカル、`git push` で ローカル → GitHub、`git pull` で GitHub → 手元、という流れをカードの移動で表現します。

---

## 主な機能

1. **ホーム** … サイトの説明と、3つの学び方への入口、進捗サマリー
2. **状況から学ぶ** … 「変更を保存したい」などの状況から、使うコマンド・理由・状態変化を表示
3. **コマンドから学ぶ** … 各コマンドの意味・使う場面・注意点・よくある勘違い・デモ
4. **シミュレーター** … 疑似ターミナルでコマンドを実行し、4エリアの変化を確認
5. **クイズで復習** … 選択式クイズ。間違えた問題は自動で復習リストに保存
6. **進捗表示** … 学習済みコマンド数・クイズ正答率・復習が必要な数を localStorage に保存
7. **ブランチを学ぶ（`git/branch.html`）** … 独立ページ。ブランチとは何か、`main` と作業ブランチの違い、分岐・合流（merge）のしくみをCSSだけの図解で解説
8. **チーム開発を学ぶ（`git/team.html`）** … 独立ページ。複数人で共有するときの運用（mainを守る原則、ブランチ→PR→レビュー→マージの1サイクル、マージ戦略の違い、コンフリクト対処、やってはいけないこと）を図解で解説

### 対応コマンド

`git status` / `git add .` / `git add <ファイル>` / `git commit -m "..."` /
`git push` / `git pull` / `git restore <ファイル>` / `git restore --staged <ファイル>` /
`git branch` / `git switch <ブランチ>` / `git switch -c <ブランチ>` /
`git log --oneline` / `git diff` / `git remote -v`

## ジャンルを増やすには

1. ルートに新しいフォルダを作る（例: `english/`）
2. その中に `index.html` を置き、共通CSSを `<link rel="stylesheet" href="../css/style.css">` で読み込む
3. ヘッダーに「← atelier」リンク（`../index.html`）を入れて戻れるようにする
4. ルート `index.html` の「ジャンルを選ぶ」に、そのジャンルのカードを1枚足す

問題形式や画面の作りは、そのジャンルに合わせて自由に設計してOKです
（共通なのは“デザインの見た目”と“ハブからの導線”だけ）。

## ローカルでの確認方法

ビルドツールは不要です。次のいずれかで開けます。

- **そのまま開く**: `index.html` をダブルクリックしてブラウザで開く
- **簡易サーバーで開く**（推奨。一部ブラウザの制約を避けられます）:

  ```bash
  # Python が入っている場合
  python3 -m http.server 8000
  # → ブラウザで http://localhost:8000 を開く
  ```

---

## GitHub Pages で公開する方法

1. このリポジトリを GitHub に push する
2. GitHub の **Settings → Pages** を開く
3. **Build and deployment** の **Source** を `Deploy from a branch` にする
4. **Branch** を公開したいブランチ（例: `main`）、フォルダを `/ (root)` に設定して **Save**
5. 数十秒〜数分で `https://<ユーザー名>.github.io/<リポジトリ名>/` に公開されます

すべて相対パスで読み込んでいるので、サブディレクトリ公開でもそのまま動きます。

## 進捗データについて

学習進捗はブラウザの `localStorage`（キー: `gvt_progress_v1`）に保存されます。

- 学習済みコマンド（コマンド詳細を開いたもの）
- クイズの回答数・正答数
- 間違えた問題の復習リスト

クイズ画面の「進捗をリセット」ボタンで消去できます。別のブラウザ・端末とは共有されません。
