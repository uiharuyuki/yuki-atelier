教科書

1. OSI参照モデル（7層）

ネットワークの通信を、役割ごとに7つの層に分けた設計モデル。ISO（国際標準化機構）が作った理論モデルです。実際の通信は次の TCP/IP モデルで動いていますが、トラブル切り分けと概念整理は OSI で話すのが業界標準です。

層名前役割代表例
L7アプリケーション層アプリが使うプロトコルHTTP, HTTPS, DNS, SMTP
L6プレゼンテーション層データ形式の変換・暗号化TLS, 文字コード, JPEG
L5セッション層通信の開始〜終了の管理セッション確立/切断
L4トランスポート層端末間の信頼性・順序制御TCP, UDP
L3ネットワーク層ネットワーク間のルーティングIP, ルータ
L2データリンク層同一セグメント内のフレーム転送Ethernet, MAC, スイッチ
L1物理層電気・光信号、ケーブルUTP, 光ファイバ, ハブ
豆知識: ISO は組織（国際標準化機構）、OSI は ISO が作った通信モデルの名前。略称の「ISO」はギリシャ語の isos（等しい）から取っているので、どの言語でも ISO で統一されています。

2. TCP/IP モデル（4層）

実際のインターネットで動いているモデル。OSI の 7層を 実装ベースで 4層に圧縮したものです。

TCP/IP層対応するOSI代表プロトコル
アプリケーション層L5〜L7HTTP, DNS, DHCP, SMTP
トランスポート層L4TCP, UDP
インターネット層L3IP, ICMP, ARP
ネットワークインターフェース層L1〜L2Ethernet, Wi-Fi
CCNA 試験では OSI の層番号で問われることが多いです。まず OSI を体に入れて、TCP/IP は「実装側の呼び方」として押さえておけば OK。

3. カプセル化とPDU（重要）

データが送信側で 各層を下るとき、ヘッダが付きます。受信側では上に行くとき ヘッダが剥がれます。データの単位（PDU: Protocol Data Unit）は層で呼び方が変わります:

層PDU名
L4 トランスポート層セグメント（TCP） / データグラム（UDP）
L3 ネットワーク層パケット
L2 データリンク層フレーム
L1 物理層ビット
「L3の PDU は？」→ パケット、みたいに層番号と PDU 名を紐付けて即答できるようにしておくと点が取れます。
